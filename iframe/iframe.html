Iframe Content

Inside an iframe, the Document.referrer will initially be set to the same value as the href of the parent window's Window.location.

The document.referrer property in JavaScript provides the URL of the document that loaded the current document. However, its usage can be limited and may not always contain the expected value. Specifically, in some cases, it may be empty or not provide the information you need.

In the context of cross-document communication, such as between a parent window and an iframe, it's often more reliable to use the postMessage API, as demonstrated in this example. This method allows for secure communication between documents and can be more explicit in terms of specifying the target origin.

Using document.referrer in this context might not be suitable because:

Security Concerns: The document.referrer value is set by the browser and can be controlled by the user or may be blocked for security reasons.

Limited Information: The referrer may not always be available or might not contain the information you need, especially if the documents are on different domains.

If you have control over both the parent and iframe documents and they are on the same domain, you might consider using the window.parent reference directly. However, be cautious about the security implications of cross-origin communication.

For secure and explicit communication between documents, especially across origins, using postMessage is generally recommended. It provides a more controlled and secure way for documents to communicate with each other.
